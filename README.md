## 🚀 Quick Start
Follow the steps below to get the project humming locally.

```bash
# 1. Clone the repository
$ git clone https://github.com/YOUR_USERNAME/audio-player-react.git
$ cd audio-player-react

# 2. Install dependencies (requires Node.js 18 or newer)
$ npm install

# 3. Launch the development server with hot reload
$ npm start
```

Once the server is up, visit **http://localhost:3000** to explore the playlist.

## 🧰 Available Commands
| Command | Description |
|---------|-------------|
| `npm start` | Runs the app in development mode using Webpack Dev Server with hot reload. |
| `npm run build` | Bundles the app for production and outputs minified assets to `dist/`. |
| `npm run deploy:github` | Publishes the production build straight to GitHub Pages in a single command. |

## 🎵 Playlist Catalog
Experience a curated mixtape of 8-bit nostalgia and cartoon anthems. Each track streams directly from the [BreatheCo Audio API](https://assets.breatheco.de/apis/sound/).

| Category | Featured Tracks |
|----------|-----------------|
| **Video Games** | Mario Castle · Mario Star · Mario Underwater · Zelda Castle · Zelda Outworld · Sonic Brain Zone |
| **Cartoons** | Flintstones · Power Rangers · The Simpsons · South Park · Thundercats · X-Men |

Extend the playlist by appending new objects to the `list` array in `src/js/component/home.js`.

## ❓ FAQ
**Why Node.js 18 or later?**  
The toolchain and dependencies leverage modern Node features and ESM compatibility that ship with Node 18+.

**Can I use my own audio files?**  
Absolutely! Replace the `url` values in `home.js` with paths to your assets (local or remote) and update the `category`/`name` fields as needed.

**How do I deploy to production?**  
Run `npm run build` to generate the production bundle, then `npm run deploy:github` to push it live via GitHub Pages.
